Chapter 1. Download the Zapatec DHTML Menu Files

1. Download the Zip File

The DHTML Menu files are bundled in a zip file. Click the download link in the Menu section on Zapatec web site and follow the instructions to download the file.

Save the file (zpmenu.zip) to your website root folder on your computer or server.

2. Extract zpmenu.zip

  1. On your computer, browse to the folder where zpmenu.zip file is saved.

  2. zpmenu.zip contains zapatec folder that holds all the files you need to create your menu.

  3. Double-click zpmenu.zip file to extract it.

Chapter 2. Create a Menu Using the Wizard

You can create a Zapatec menu with or without using the wizard.

1. What Does the Wizard Do?

The wizard generates the code for the exact kind of the menu you need. The Graphic User Interface (GUI) allows you to select the menu features visually. Once you have made your selections, the wizard creates the menu-generating code that you can insert in your HTML document, form or application with a click of a button.

2. How to Start the Wizard

Follow the instructions in section 1.1. to download the Menu files.

  1. Navigate to your DHTML Menu folder (zpmenu).

  2. Open index.html in your Web browser.

  3. Click the Wizard icon in the left menu.

3. Using the Wizard

In the wizard window, click the Next or Previous button to move through the tabs, or choose a specific tab from the tabbed menu on the top of the window. Position the pointer over the question marks for additional information about various options.

The first tab makes you familiar with the wizard and explains how to use it. Here, you may set the number of sub-menus to be displayed in the main window as well as the number of items in each sub-menu. The menu is defined as a collection of HTML <UL> and <LI> sections, which you can edit in your HTML-editing program.

4. Pane - Introduction

This pane allows you to define the basic framework for your menu.

  • Number of Sub-Menus - Enter the number of top-level menu items or choose the appropriate number from the drop-down list. For example, to create a menu consisting of Home (1), Products (2), Support (3), About (4), My Account (5), choose 5.

  • Number of Items - Enter or choose from the drop-down list the number of items to be displayed in each sub-menu. You can add and remove items in the Contents pane.

  • Vertical Menu - By default, the wizard generates horizontal drop-down menus. Select this check box to position your menu vertically. The menu items, in this case, will pop out to the right of the menu.

  • Click Menu - Normally, the drop-down menus open when you position a pointer on the item. Select this check box to have the menus drop when user clicks the item. This option is only applicable for the top-level menu items.

  • Trigger Menu - By selecting this option, you can associate opening of your menu with a mouse-click or key-press event. You can choose either of these options from the drop-down list:

    • Right Mouse Button

    • Left Mouse Button

    • Any Mouse Button

    • Keyboard

      • Trigger Key Scan Code - This is the decimal scan code of the associated key menu. For example, 113 is the key code for the F2 key. Required for the keyboard trigger event.

    • Trigger Object - This is the optional object ID associated with the menu. For example, it could be DIV inside which users can click to display the menu. Leave this field blank to associate the menu with the entire document.

  • Keep Track of Previous Menu Location - If this option is selected when users click an item, the previous location of the menu is saved in a cookie. When the menu is redisplayed on the target page, this item and the path to it will be highlighted.

    • Expand Previous Location: Menu will open expanded to the saved location.

    • Cookie Name - Enter the name for the cookie that will store the saved menu location. If you have multiple menus on a page, use a different cookie for each menu.

5. Pane - Contents

This pane builds on the menu framework that you defined in the Introduction pane. You can modify the contents of the menu items using Add, Change and Delete commands. You can also change sub-menus. Since the menu is built using standard HTML lists, you can create the actual menu contents in your editor, in a graphical user interface (GUI) or have your server generate the menu items automatically from a database.

Click a menu item for making changes. The selected menu item will be marked by a red border.

  • Item Text - Enter the text to appear in the menu item.

  • Item Link - Insert the absolute or relative link for the menu item. For example, a link to Zapatec home page would be http://www.zapatec.com.

  • Link Target - Edit the target for this link. For example, if you want a new page to open for the Item Link, enter _new for the Link Target. Valid link targets are:

    • _top

    • _bottom

    • _blank

    • _new

    • your own custom target name.

  • Insert Item Before - Inserts a new menu item before the selected item. In the Item Text field enter a name for the new item, and in the Item Link field insert the a link to the item.

  • Insert Item After - Inserts a menu item after the selected menu item. Enter a name for the new item in the Item Text field, and in the Item Link field provide a link to this item.

  • Remove item - Removes the selected item.

  • Add Subtree - Adds a sub-menu under the selected menu item.

  • Add Icon - Adds an icon for this menu item. Select Add Icon to open a dialog box containing a list of icons from the Zapatec Icon Gallery. Select an icon and click OK. Otherwise, in the Icon URL field enter a path to the icon file you want to use and click OK. If the menu item already has an icon assigned, the Remove Icon command will be displayed instead of Add Icon.

6. Pane - Look and Feel

This pane allows you to customize the look and feel of your menu. You can select any of the multiple predefined themes, change the look and feel of the entire menu, sub-menu, or a single menu item. Click the menu or menu item that you want to edit. A red border appears around the element that you are currently editing.

The property links activate a window where you can edit the menu and select the menu item properties. You can observe how the menu changes when you choose a property. This enables you to get a real-time view of the presentation layer evolution. If the modifications did not produce the desired effect, you can reset the item to its previous state using the Reset button in the property window.

These are the editable categories that govern the look and feel of a menu:

  • Theme - Use the drop-down list to select a theme you want to apply to your menu. The theme controls the menu color scheme, design elements, and layout.

  • Menu Properties - Customize the appearance of all menu items.

  • Item Properties - Customize the appearance of the selected menu item only.

  • Sub-Menu Properties - Customize the appearance of the selected sub-menu.

In the Properties windows you can change the following:

  • Font

    • Font Family

    • Font Style Bold

    • Font Style Italic

    • Font Size in pixels

  • Style

    • Background Color

    • Background Image URL

    • Text Color

  • Highlighted Style

    • Background Color

    • Background Image URL

    • Text Color

  • Border

    • Width in pixels

    • Style - solid, dashed, ...

    • Color

  • Dimensions & Align

    • Width in pixels

    • Height in pixels

    • Text align - default, left, right, center, justify

    • Vertical align - default, top, middle, bottom, baseline, sub, super, text-top, text-bottom.

7. Pane - Effects

This pane allows to you choose the special effects for the menu.

  • Scroll Menu - Select this check box to make your menu scroll and stay in the same original position on the page while scrolling the page up or down.

  • Draggable Menu - If this check box selected, the menu can be dragged across the window when clicked.

  • Glide Sub-Menus - Select this check box to make your menu gradually glide in and out on click or hover instead of appearing and disappearing immediately. This option can be combined with the fade option.

  • Fade Sub-Menus - When this check box is selected, the menu fades in and out gradually if clicked or hovered instead of appearing and disappearing immediately. Can be combined with glide.

  • Slide Sub-Menus - By selecting this check box, you may enable your menu to slide in and out. A sub-menu slides down while expanding and up while collapsing. The bottom of the sub-menu appears first and expands from the bottom to top.

  • Wipe Sub-Menus - Select this check box to have your menu wipe in and out. The width and height of the sub-menu expand and collapse at the same rate.

  • Unfurl Sub-Menus - If this check box selected, the menu unfurls in and out. The width expands first and then the height.

  • Drop Shadow Sub-Menus - Select this check box to make the right and bottom side of the sub-menu cast a shadow.

  • Advanced Options:

    • Animation Speed - Choose the relative animation speed for the effects listed above. You can select either of the following: very slow, slow, normal, medium, fast or very fast. Normal is set by default.

    • Show Delay - Using this option you can set the speed at which sub-menus are being expanded. Select the value you need from the drop-down list.

    • Hide Delay - With the help of this option you can set the speed at which sub-menus are being collapsed. In the drop-down list, select the appropriate value.

8. Pane - Get Your Code

The wizard displays the code for your menu. Click Select All and press Ctrl-C to copy the code to the clipboard. Open your editor, paste and save it.

The paths in the code assume that the menu file is in the same folder as the zpmenu folder, not in the zpmenu folder itself. If you want move your file to a different folder, adjust the paths accordingly.

Chapter 3. Quick Startup (Without Wizard)

This section describes how to create menus in HTML and Javascript.

If you are not using the wizard, you may follow the instructions in Section 1.1. to download and extract the Menu files.

1. Set Up Your Menu

In your web editor (Dreamweaver, UltraEdit, etc.), open the web page where you want the menu to be placed. Position a cursor before the ending </head> tag. Paste the following style path and script paths before the ending </head> tag:

<!-- Javascript Zapatec utilities file -->  
<script type="text/javascript" src="zapatec/utils/zapatec.js"</script>>
<!-- Javascript file to support menu -->
<script type="text/javascript" src="zapatec/zpmenu/src/zpmenu.js"></script> 
                                                                             
<!-- Optional Javascript file for keyboard shortcuts --> 
<!-- Not needed if you don't use keyboard shortcuts -->
<script src="zapatec/zpmenu/src/menu-kbd.js" type="text/javascript"></script>

Keyboard shortcuts are demonstrated in the Shortcuts Demo.

1.1. Path Warning

If the web page containing the menu is saved in the zpmenu folder, these lines will work without any changes. If, however, you saved your web page in another location (for example, directly under your website root folder), you need to adjust the path for these files.

1.2. Upgrade Warning

If you have been using a version of Zapatec menu that is older than 2.0 and you want upgrade to the current version, you need to adjust the header files.

The older menu.js is no longer supported; zpmenu.js is now used instead. To upgrade, replace the old line:

<script type="text/javascript" src="../src/menu.js"></script>

with the following lines:

<script type="text/javascript" src="zapatec/utils/zapatec.js"></script>
<script type="text/javascript" src="zapatec/zpmenu/src/zpmenu.js"></script>

In addition, the use of the LINK element for CSS inclusion in pre-2.0 versions is no longer supported. For example, if you used the office_blue.css in the HEAD of the HTML page your pre-2.0 version as in:

<link href="../themes/office_blue.css.css" rel="stylesheet" type="text/css"/>

you should delete this line. Instead, set the CSS using the theme option in the Zapatec Menu constructor. For example:

var menu=Zapatec.Menu({theme:'office_blue'})

See the theme Configuration Option section for details.

2. Create the Menu

First, define the data for the menu and then create the menu object.

2.1. Define the Data

The menu data is defined as a nested HTML list. The items can be in unordered lists <UL> or ordered lists <OL>. The top <UL> must have an id parameter that will be used for creating the menu object.

 <!-- The HTML for the menu-->
 <ul id="myMenu">
  <li>
  menu 1
  <ul>
   <li> Item 1 </li>
   <li> Item 2 </li>
   <li> Item 3 </li>
   <li> Item 4 </li>
  </ul>
  </li>
  <li>
  menu 2
  <ul>
   <li> Item 1 </li>
   <li> Item 2 </li>
   <li> Item 3 </li>
   <li> Item 4 </li>
  </ul>
  </li>
  <li>
  menu 3
  <ul>
   <li> Item 1 </li>
   <li> Item 2 </li>
   <li> Item 3 </li>
   <li> Item 4 </li>
  </ul>
  </li>
  <li>
  menu 4
  <ul>
   <li> Item 1 </li>
   <li> Item 2 </li>
   <li> Item 3 </li>
   <li> Item 4 </li>
  </ul>
  </li>
 </ul>

2.2. Create the Menu

The easiest way to create the menu is to do this when the HTML finishes loading. 

<body onload="new Zapatec.Menu('myMenu')">

An alternative is to place the following code after the section in which the data for the menu is defined. Do not place it before that section, because Javascript will not be able to find the menu definition and will generate an error.

<script type="text/javascript">
 new Zapatec.Menu("myMenu");
</script>

3. Copy Files to the Web Server

Using your editing or FTP program, copy or move your web page and the entire tree folder to the root of your website.

4. Test the Menu Application

Using your web browser, navigate to the web page created to include the Zapatec HTML Menu. You have set up the most basic version of the Zapatec HTML Menu! Now, you can change this highly adaptable application with the help of multiple features provided.

5. Load Menu from External File

An alternative to defining the data within the page is to load the menu data from an external HTML file.

To do this, use the source , sourceType , container , and theme options to load the menu data.

6. Custom Icons and Anchor Links

6.1. Custom Icons

You can add custom icons to each menu item by inserting image <IMG> tags inside the item HTML list item tags:

<li>
 <img src="icon.gif" alt="An icon" />
 Our item with an icon
</li>

6.2. Anchor Links

Each menu item may possess an anchor link to another document. Here is an example that creates a menu item to point to your home page:

<li>
 <a href="home.htm">Home Page</a>
</li>

Here is an example that creates a custom icon (home.gif) and a link to your home page (home.htm):

<li>
 <img src="home.gif"><a href="home.php">Home Page</a>
</li>

7. Multiple Levels

The following is an example of a menu with three levels of sub-menus. The start of a sub-menu is contained in <UL> tags. 

<li><a href="#">Many Levels</a>
<ul>
 <li><a href="#">Level-1a</a></li>
 <li><a href="#">Level-1b</a> <!-- Level-1b Start sub-menu -->
 <ul>

  <li><a href="#">Level-2a</a></li>
  <li><a href="#">Level-2b</a> <!-- Level-2b Start sub-menu -->
  <ul>

   <li><a href="#">Level-3a</a></li>
   <li><a href="#">Level-3b</a> <!-- Level-3b Start sub-menu -->
   <ul>

    <li><a href="#">Level-4a</a></li>
    <li><a href="#">Level-4b</a></li> <!-- Lowest Level -->

   </ul> <!-- Level-3b End sub-menu -->
   </li> <!-- Level-3b End menu item -->

  </ul> <!-- Level-2b End sub-menu -->
  </li> <!-- Level-2b End menu item -->

 </ul> <!-- Level-1b End sub-menu -->
 </li> <!-- Level-1b End menu item -->

</ul> <!-- Many Levels sub-menu -->
</li> <!-- Many Levels -->

Chapter 4. Configuration Options

As mentioned before, you can create a menu using the following syntax:

<body onload="new Zapatec.Menu('myMenu')">

You can also use a set of parameters to control the menu behavior when being created: 

<script type="text/javascript">
 var myMenu = new Zapatec.Menu({source: 'myOtherMenu', hideDelay:500, onClick:true})
</script>

This specifies that the menu should use the "myOtherMenu" list, sub-menus should hide in 500 Msec,.5 seconds after the cursor has been moved away from them, and that the top menu should only display when clicked.

Another part of this section describes the menu configuration parameters.

1. Configuration Parameters

1.1. source (string)

Use the source option along with the sourceType and container options to retrieve a menu from an external HTML file instead of creating it with the wizard or defining the menu data on the page itself.

The source specifies the source file of the external menu data.

The following example receives the menu data from an external file: 

<div id="menuContainer"></div>

<script type="text/javascript">

  var objMenu = new Zapatec.Menu({
    theme: 'rounded',
    source: 'menu_source.html',
    sourceType: 'html/url',
    container: 'menuContainer'
  });

</script>

1.2. sourceType (string)

Use the sourceType along with the source , container and theme options to set the type of the menu data external source.

The sourceType option is used with the source option to specify how the menu source should be processed.

Valid source types are html and html/url. The default is html.

1.3. container (string OR object)

The container option specifies the element or ID of the element that contains the menu data.

Required when sourceType is other than html.

If sourceType is html and container is not specified, menu source element will be replaced with the menu.

1.4. theme (string)

The theme option defines the CSS used to present the menu.

The string argument is the theme name:

theme : 'winxp2'

or the absolute or relative path to the theme file:

theme : 'themes/winxp.css'

The theme name is case-insensitive. Several themes contain in the themes folder.

If the theme option is not specified, the default.css file in the themes folder is used.

To use your own CSS throughout the enterprise without specifying the theme option for each page, you can rename your CSS file "default.css" and move it to the theme folder.

If you have several menus on the page, you can apply different themes for each menu. See the Two Menus Demo.

1.5. showDelay (integer, default 0)

This is the interval, expressed in milliseconds, between the time when the pointer is hovered over the menu item and the menu itself appears.

See the Show and Hide Speeds Demo.

1.6. hideDelay (integer, default 500)

This is the interval, expressed in milliseconds, between the time when the pointer is moved away from the menu item and the menu disappears. Decreasing the interval below the default value can cause the menu to appear "jerky".

See the Show and Hide Speeds Demo.

1.7. onClick (boolean, default "false")

When onClick is false, by default, drop-down menus display on hover. When onClick is true, the menus drop when clicked. This is how most desktop applications behave.

Choosing this option has only effect on the top-level menu items.

See the Click Demo.

1.8. vertical (boolean, default "false")

By default, menus are displayed horizontally. This means that the top level of the menu is a horizontal bar and the sub-menus display below. Set the vertical option to true for a vertical menu in which the top level items display in a column and the sub-menus open to the right.

See the Vertical Demo.

1.9. zIndex (integer, default 0)

This is the display order for the menu as it relates to other menus. The higher the zIndex, the higher this menu priority. For example, a menu with zIndex of 10 will display on top of a menu with zIndex 9.

See the Two Menus Demo.

1.10. rememberPath (boolean or string, default false)

Set this option to true to keep track of the previous menu location visited by the user. rememberPath uses a cookie to read and write the previous menu location. Valid values for the argument are:

  • true - Keep track of location.

  • false - Do not keep track of previous location.

  • "expand" - Keep track of location and re-open expanded to this location when the menu is accessed.

The CSS class name zpMenuPath is associated with the menu item whose path has been remembered. The style of all menu items leading to the selected menu item will be highlighted, based on zpMenuPath.

This feature works in an application across multiple pages in which identical menus appear on every page, as long as all the menus are initialized in the same order on every page.

rememberPath is optional if the pathCookie flag option is not __zp_item.

See the Two Menus Demo.

1.11. pathCookie (string, default ’__zp_item’)

The string argument is the path to the cookie that is used to keep track of previous menu location.

Use this option with or instead of the rememberPath option when you need to specify which cookie contains path information about the user last location on this menu.

The pathCookie option is required when you have multiple menus on a page and need to remember the unique path to the previous location of each menu.

If the rememberPath option is not false and the pathCookie option is not set, then pathCookie defaults to __zp_item.

See the Track Menu Demo.

1.12. triggerEvent (string, default null)

This is the name of the event that triggers the display of the menu. This option is used to implement context-sensitive menus.

Valid trigger names are keydown, keyup, keypress, mousedown, mouseup, and click.

See the Context Sensitive Demo.

See also triggerObject and triggerKey .

1.13. triggerObject (string, default null)

This is the ID of the HTML element associated with the menu that is triggered by the triggerEvent. Ignored if triggerEvent is not set.

If triggerEvent is set and triggerObject is not defined, document.all is used.

See the Context Sensitive Demo.

See also triggerEvent and triggerKey .

1.14. triggerKey (integer or string, default null)

This is the value of the specific keystroke or mouse button that will trigger the display of the menu. This option is used to implement context-sensitive menus.

The triggerEvent must be set for a keystroke to trigger the menu. In this case, the triggerEvent is keydown, keyup, or keypress, and you use the keyboard scan code as a value of the keystroke for the triggerKey.

The triggerKey is optional for a mouse triggerEvent. By default, the right mouse button triggers the menu. If the triggerEvent is mousedown, mouseup, or click, the valid values for the triggerKey are right, left, or both.

See the Context Sensitive Demo.

See also triggerEvent and triggerObject .

1.15. top (string or int, default null)

This is the absolute offset, in pixels, of the menu from the top of the document.

See the Absolute Position Demo.

1.16. right (string or int, default null)

This is the absolute offset, in pixels, of the menu from the right of the document.

See the Absolute Position Demo.

1.17. bottom (string or int, default null)

This is the absolute offset, in pixels, of the menu from the bottom of the document.

See the Absolute Position Demo.

1.18. left (string or int, default null)

This is the absolute offset, in pixels, of the menu from the left of the document.

See the Absolute Position Demo.

1.19. defaultIcons (string)

If set, all tree items will get an additional TD element containing that string in the class attribute. This helps you to include custom default icons without specifying them as IMG tags in the menu.

1.20. onInit (function)

The referenced function will be called when menu is initialized.

This function can be used to disable certain items, etc.

1.21. preventDoubleCall (boolean, default false)

Use this option set to true to prevent clicks on the menu after the menu item was clicked.

After the user clicks a menu item, it becomes disabled, so that the user cannot click it again and force double form to submit.

2. Special Effects Parameters

These parameters add special effects to your menu. While multiple effects can be used, not all combinations of effects work properly together.

2.1. dropShadow (integer, default 0)

This option makes the right and bottom sides of the sub-menu cast a shadow.

This is one of our most popular effects. It gives the sub-menu a 3D appearance. The integer sets the opacity of the shadow. For example, to set the percentage of shadow opacity to 25%, you should specify the value of 25.

See the Drop Shadow Demo.

2.2. scrollWithWindow (boolean, default "false")

If this option is true, when the user scrolls down the page, the menu scrolls and stays in the same user-defined position on the page.

See the Scroll With Window Demo.

2.3. drag (boolean, default "false")

When this option is true, the user can click anywhere on the menu and drag it.

See the Draggable Menu Demo.

2.4. fade (boolean, default "false")

When this option is true, the menu fades in (increases in opacity) on mouse over and fades out (decreases in opacity) on mouse out.

See the Special Effects Demo.

2.5. glide (boolean, default "false")

When this option is true, the menu glides in (increases clip) on mouse over and glides out (decreases clip) on mouse out.

See the Special Effects Demo.

2.6. wipe (boolean, default "false")

When this option is true, instead of appearing and disappearing immediately, the menu wipes in and out. The width and height of the sub-menu expand and collapse at the same rate.

2.7. slide (boolean, default "false")

When this option is true, instead of the menu appearing and disappearing immediately, it slides in and out. The sub-menu slides down while expanding and slides up while collapsing. The bottom of the sub-menu appears first and expands from the bottom to top.

2.8. unfurl (boolean, default "false")

When this option is true, instead of appearing and disappearing immediately, the menu unfurls. The width expands before the height.

2.9. animSpeed (integer, default 10)

This option sets the speed at which the special effect frames render. For example, a speed of 10 for the fade effect during mouse over will show 10 percent more opacity each 50 milliseconds until 100 percent opacity is reached. A value of 10 for the glide effect will show 10 percent of the menu on 1st pass, 20 percent on 2nd, etc. until 100 percent of the menu is displayed.

Chapter 5. Menu Tips

1. Window Borders

The menus automatically display where they fit best, based on the window borders. For example, if the menu is aligned with the rightmost window border, the sub-menus display to the left.

2. Disable/Enable Menu Items

You can dynamically disable and enable menu items.

See the Disabled Demo.

Use menu.itemDisable(string) to disable or menu.itemEnable(string) to enable a menu item. The argument is the ID of the item to be enabled or disabled.

There are several cases in which it might be desirable to disable a menu item. For example, you might want to disable the menu item that points to the current web page. Or, it might be appropriate to disable a shopping cart product if it is out of stock, but you still want to show the product. A good way to automate this is to have the LI id be the SKU number. If the SKU of the item is out of stock, call menu.itemDisable(SKU). 

var menuSku = new Zapatec.Menu('menuSku', {});
menuSku.itemDisable('112233');  // LI id of menu item product whose SKU is 112233
var menuNav= new Zapatec.Menu('menuNav', {});
menuNav.itemDisable('home');    // LI id of menu item that points to home.html

3. Target for Menu Item Links

The link target for a menu item can have different link target property values. The link target specifies where the linked data appears.

To specify a link target, add the TARGET attribute to the anchor of the LI item that describes the menu item. The following example displays the data in a new window:

<li><img border='0' src='../themes/icon/icon_cut.gif'> <a href="http://www.zapatec.com" target="_blank">Cut</a>  </li>

The next example displays the data in a frame named "frame": 

li><img border='0' src='../themes/icon/icon_print.gif'> <a href="http://www.zapatec.com" target="frame">Print...</a></li>

The following values are valid for a link target of a menu item:

  • target="FrameName" - Opens the page in the frame named FrameName.

  • target="_blank" - Opens the page in a new window.

  • target="_new" - Opens the page in a new window.

  • target="_top" - Opens the page in the same window as the menu item.

See the Target Demo.

4. Context-Sensitive Menus

You can specify that a menu must be activated only in a certain section of the screen and only in response to a certain keystroke or mouse click.

It would be better to allow users to press F1 and give them a menu of Help items. And on the same web page they could right-click in a certain DIV section to see a menu of the products you sell on your web site. You can enable this behavior with context-sensitive menus.

Context-sensitive menus rely on three configuration options: triggerObject , triggerEvent , and triggerKey .

When the triggerEvent option is set for a menu, the menu appears only when that event happens. The event is typically a mouse or keyboard action by the user.

The triggerObject provides the context in which the menu is triggered. It is the ID of the element on the web page that is associated with the menu content. For example, if you are providing a menu of Help items related to elements displayed in a DIV, the ID of the DIV would be the triggerObject and the menu would appear in or near the DIV when it is activated.

If the triggerEvent is a mouse event, the default action is a mouse right-click. If you want the triggerEvent to respond to another mouse action, such as a left-click or a click of either mouse button, or to a keystroke, set the triggerKey.

See the Context Sensitive Demo for examples.

The following script illustrates various scenarios for context-sensitive menus.

<script type="text/javascript">
   // activated by Right Click anywhere outside other trigger objects.
   var menuAll = new Zapatec.Menu('myMenu', {triggerEvent: 'mouseup'});
   // activated on F1 (value is 112)
   var menuF1 = new Zapatec.Menu('myMenuEnter', {triggerEvent: 'keydown', triggerKey: 112});
   // activated on right click
   var menuRight = new Zapatec.Menu('myMenuRight', {triggerEvent: 'mouseup', triggerObject: document.getElementById('area1')});
   // activated on left click
   var menuLeft = new Zapatec.Menu('myMenuLeft', {triggerEvent: 'mouseup', triggerKey: 'left', triggerObject: document.getElementById('area2')});
   // activated on left OR right click
   var menuBoth = new Zapatec.Menu('myMenuBoth', {triggerEvent: 'mouseup', triggerKey: 'both', triggerObject: document.getElementById('area3')});
</script">

5. Keyboard Shortcuts

A user can navigate Zapatec menus using keyboard shortcuts.

To enable keyboard shortcuts, you must include menu-kbd.js in the headers. See Path Warning section for details.

When keyboard shortcuts are enabled, for each item in the menu having an underlined character, a user can press Alt + the underlined character to perform the menu action or expand its submenu.

For each active submenu item having an underlined character, a user can either press Alt + the underlined character or just the underlined character to perform the menu action or expand its submenu.

In an active menu item, a user can navigate through the menu by pressing the arrows, ESC, or ENTER keys or Space. A user can traverse menu items by pressing the TAB key repeatedly.

See the Shortcuts Demo.

6. Long Auto Scrolling Menu

When a sub-menu consists of more items than can be displayed on a page, a user can view the hidden items by moving the mouse to the up arrow at the top of sub-menu or to the down arrow at the bottom of sub-menu.

See the Long Auto Scrolling Menu Demo.

7. Absolute Position

If the top, right, bottom or left options are set, the top_parent DIV is positioned absolutely. The options values are assigned to corresponding CSS properties of the top_parent DIV.

You should set either the top or bottom option and the right or left option for drag and scroll menus instead of putting them inside absolute-positioned DIVS. Otherwise, the menu may be in an incorrect position.
